# Переделка проекта: Шугнов Голд → NETS (IT системный интегратор)

## Концепция

**NETS** — системный интегратор: проектирование и внедрение IT-инфраструктуры, сетей, ЦОД, информационной безопасности, ПО и сервисной поддержки для корпоративных клиентов. Штат ~260 человек.

Все 8 вкладок сохраняются, но их смысл адаптируется под IT-компанию.

## Адаптация смысла вкладок

| Вкладка | Было (золотодобыча) | Стало (IT-интегратор) |
|---|---|---|
| По локациям | Карьер, ЗИФ, базы | Офисы: HQ, R&D-центр, NOC/SOC 24×7, Сервисный хаб, Склад/логистика, Региональные офисы |
| По производственному циклу | Вскрыша → промывка → аффинаж | Pre-sale → Архитектура → Поставка → Внедрение → Интеграция → Эксплуатация/поддержка |
| По иерархии | Гендир → Главинженер … | CEO → CTO, COO, CFO, CSO (безопасность), CHRO, CRO (продажи) |
| Штатное расписание | Те же должности, оклады TJS | IT-должности, оклады в USD (диапазон ~$800–$8000) |
| HR аналитика | Текучесть по цехам | Текучесть/найм/grade-mix по IT-департаментам |
| Глоссарий | Горные термины | IT-термины: SLA, SOC, SD-WAN, MPLS, SIEM, Kubernetes, ITIL, ЦОД Tier III и т.д. |
| Карта | Пойма р. Шугнов | Карта офисов (Душанбе HQ + регионы), без оф. геоданных |
| Оргсхема | Блочная иерархия | То же, новые должности и департаменты |

```

│   ├── Проектный офис (PMO) (10)

│   ├── Аудит и compliance (6)
│   └── Физ. безопасность и режим (7)
├── CRO — Коммерческий блок (~20)
│   ├── Продажи корп. клиентам (10)

│   └── Партнёрства (вендоры) (5)
├── CFO — Финансы и администрирование (~10)
│   ├── Бухгалтерия (5)

```
Итого ~266 чел.

## Технические замечания

- Минимальные правки логики: переиспользую существующие компоненты как есть, меняю только данные и подписи
- ID департаментов/локаций поменяются — старый localStorage (штатка) станет невалидным; это ок, сбросится при первом редактировании
- Карта без реальных координат — статическая иллюстрация офисов в виде сетки/блоков с подписями
- Цветовая тема остаётся янтарной (yellow/amber), но допускаю переход на сине-фиолетовую IT-палитру — уточню после плана только если потребуется

## Что НЕ делаю в этом шаге

- Логин/авторизация (откладываю — это отдельный запрос)
- Реальные географические карты с API
- Загрузка фото в глоссарий — оставляю как есть
