## Что добавим

Новую вкладку **«Глоссарий»** (иконка `BookOpen`) рядом с существующими — интерактивный справочник терминов золотодобычи с фото-плейсхолдерами, которые можно заменить своими изображениями.

## Состав терминов (≈25 базовых, легко расширяемых)

Сгруппированы по категориям:
- **Продукт / материал**: ШЛИХ, ШЛИХОВОЕ ЗОЛОТО, КОНЦЕНТРАТ, ЭФЕЛЯ (хвосты), ПЕСКИ, ВСКРЫША, СПЛАВ ДОРЕ
- **Оборудование**: ППМ-5, ШОУ, БУТАРА, ШЛЮЗ, ОТСАДОЧНАЯ МАШИНА, КОНЦЕНТРАТОР, МАГНИТНЫЙ СЕПАРАТОР, ПЛАВИЛЬНАЯ ПЕЧЬ, ЭКСКАВАТОР, БУЛЬДОЗЕР, САМОСВАЛ
- **Процессы**: ВСКРЫШНЫЕ РАБОТЫ, ПРОМЫВКА, ДОВОДКА, АФФИНАЖ, АМАЛЬГАМАЦИЯ (упомянуть запрет), ОПРОБОВАНИЕ
- **Геология / участок**: РОССЫПЬ, ПОЙМА, ТЕРРАСА, ПЛОТИК, МАРКШЕЙДЕРИЯ, КАРЬЕР, ПОЛИГОН

## Интерфейс

- Сетка карточек (3 кол. на десктопе, 2 на планшете, 1 на мобильном)
- Шапка: строка поиска по названию/описанию + чипсы-фильтры по категориям + переключатель «Сетка / Список»
- Карточка: фото (16:9, плейсхолдер если не загружено), название крупно, категория-бейдж, краткое определение (1–2 строки)
- Клик по карточке → модальное окно с большим фото, полным описанием, синонимами, единицами измерения (где применимо), бейджами связанных отделов и этапов цикла с кнопками «Открыть во вкладке "По локациям/циклу"»
- Кнопка **«Загрузить фото»** на каждой карточке: открывает file picker, изображение сохраняется как data URL в `localStorage` (ключ `shugnov.glossary.photos.v1`) — без бэкенда, фото переживают перезагрузку
- Кнопка «Удалить фото» в модалке

## Связь с оргструктурой

В `src/data/glossary.ts` у каждого термина опциональные поля `relatedDeptIds[]` и `relatedStageIds[]`. Модалка показывает бейджи, клик переключает `view` и `selectedDept` в `Index` через существующий механизм `onSwitchView`. Также расширим `DepartmentDrawer` опцией «Открыть в Глоссарии» (где термин связан с отделом).

## Технические детали

- Новые файлы:
  - `src/data/glossary.ts` — массив `GlossaryTerm { id, term, category, short, full, synonyms?, units?, relatedDeptIds?, relatedStageIds? }` + категории с цветами
  - `src/components/org/GlossaryView.tsx` — сетка, поиск, фильтры, локальное состояние выбранного термина
  - `src/components/org/GlossaryCard.tsx` — карточка с upload
  - `src/components/org/GlossaryModal.tsx` — диалог с деталями и кросс-навигацией
  - `src/hooks/useGlossaryPhotos.ts` — хук работы с `localStorage` (get/set/remove по id, лимит ~1.5 МБ на фото, JPEG-сжатие через `<canvas>` перед сохранением чтобы не упереться в квоту)
- Изменения:
  - `src/routes/index.tsx` — добавить `"glossary"` в `OrgView`, кнопку в `ViewToggle` (иконка `BookOpen`), рендер `<GlossaryView />`
  - `src/components/org/DepartmentDrawer.tsx` — добавить `"glossary"` в тип, кнопка перехода
- Используем существующие shadcn компоненты: `Dialog`, `Input`, `Badge`, `Button`, `Card`, `ScrollArea`
- Стилистика: согласована с текущей светлой темой (`bg-white`, акценты `yellow-500/amber-600`)
- Никакого бэкенда, без Cloud — все данные статика + `localStorage`

## Что НЕ делаем сейчас

- Не генерируем AI-картинки (по вашему запросу — оставляем место для ручной загрузки)
- Не подключаем серверную storage — фото локально в браузере; при необходимости позже мигрируем на Lovable Cloud Storage
